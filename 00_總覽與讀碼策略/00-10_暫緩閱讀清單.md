## 所屬章節

`00_總覽與讀碼策略`

## 本節導讀

這篇是 View UI Plus 源碼閱讀時的「暫緩閱讀清單」。讀大型前端元件庫時，真正困難的地方不只是找到入口，而是知道什麼時候不要繼續追。很多支線本身有價值，但如果在第一輪過早深入，會讓主流程失焦。

本篇要解決的是：

1. 哪些區域第一輪應該先暫緩？
2. 暫緩和不讀、未解問題、已完成有什麼差別？
3. 一個支線要如何記錄，之後才找得回來？
4. 什麼條件下可以恢復閱讀暫緩項？
5. 如何避免大型元件、外部庫、dist 產物、歷史設定把第一輪讀碼拖散？

本篇的核心觀念是：

> 暫緩不是放棄，而是保護主線。  
> 一個合格的暫緩項，必須留下暫緩理由、目前證據、回來條件與下一次第一入口。

如果 `00-09_問題索引.md` 負責管理「還需要回答的問題」，那本篇負責管理「已判斷現在不追、但日後可能要回來追的支線」。

## 建議回查情境

當你遇到以下情境，可以回到這篇：

- 追源碼時分支越來越多，不確定要不要繼續。
- 讀 Select、DatePicker、Table 這類大型元件時，主流程被細節打斷。
- 想知道 `dist/`、外部庫、歷史設定、完整 examples 是否第一輪要讀。
- 某個問題有價值，但不影響目前章節通關。
- `00-09_問題索引.md` 中的問題已判斷不應該本輪處理。
- 第二輪閱讀時，需要找回之前暫緩的入口。

## 0. 五分鐘使用方式

如果讀碼時遇到支線，先用這個流程判斷。

```text
遇到支線
  -> 問：它是否阻塞當前主流程？
  -> 阻塞：留在當前筆記或 00-09 問題索引
  -> 不阻塞：判斷是否日後仍有價值
  -> 有價值：放入本篇暫緩清單
  -> 無價值：不收錄
  -> 補暫緩理由、目前證據、第一入口、回來條件
```

最小暫緩記錄格式：

| 欄位 | 必填原因 |
|---|---|
| 暫緩項 | 必須明確指出支線名稱 |
| 所屬階段 | 方便連回 [00-08_學習里程碑.md](./00-08_學習里程碑.md) |
| 暫緩理由 | 說明為什麼現在不追 |
| 目前已確認 | 防止下次從零開始 |
| 第一入口 | 下次恢復時直接打開 |
| 回來條件 | 防止永遠暫緩 |
| 狀態 | 區分暫緩中、已恢復、已完成、作廢 |

快速判斷：

| 狀況 | 放哪裡 |
|---|---|
| 本輪必須回答，否則主流程斷掉 | 當前章節筆記或 [00-09_問題索引.md](./00-09_問題索引.md) |
| 有疑問但還不知道入口 | [00-09_問題索引.md](./00-09_問題索引.md) |
| 已知道入口，但現在不該追 | 本篇 |
| 已經完成追蹤，有明確結論 | 回填章節筆記，必要時保留在本篇已完成區 |
| 不影響學習、沒有明確回來價值 | 不收錄 |

## 1. 暫緩清單的角色

暫緩清單不是垃圾桶，也不是拖延清單。它只收納「已判斷有價值，但現在追會破壞主線」的內容。

| 類型 | 意義 | 應放位置 |
|---|---|---|
| 未解問題 | 還不知道答案，且需要補證據 | [00-09_問題索引.md](./00-09_問題索引.md) |
| 暫緩支線 | 知道有價值，但現在不追 | 本篇 |
| 已完成結論 | 已有證據和答案 | 對應章節筆記 |
| 作廢想法 | 前提錯誤或沒有追蹤價值 | 不保留，或在維護記錄中標明作廢 |
| 全庫摘要 | 跨章節問題摘要 | [問題回查索引.md](../問題回查索引.md) |

合理推論：第一輪源碼閱讀最重要的是建立主線閉環。暫緩清單的存在，是為了讓你放心停止支線，而不是把每個細節都硬塞進當前章節。

## 2. 暫緩生命週期

每個暫緩項都應該有狀態變化。

```text
候選支線
  -> 暫緩中
  -> 可恢復
  -> 已恢復
  -> 已完成

特殊分支：
  -> 轉回問題索引
  -> 作廢
```

| 狀態 | 意義 | 下一步 |
|---|---|---|
| 候選支線 | 剛發現，還沒判斷是否暫緩 | 判斷是否阻塞主線 |
| 暫緩中 | 已決定本輪不追 | 等待回來條件 |
| 可恢復 | 回來條件已達成 | 建立讀碼任務卡 |
| 已恢復 | 已開始重新追這條支線 | 移到問題索引或章節筆記追蹤 |
| 已完成 | 已補完結論並回填 | 保留摘要或移出清單 |
| 轉回問題索引 | 暫緩項恢復後變成具體問題 | 移到 `00-09` 建問題卡 |
| 作廢 | 已無追蹤價值 | 標明原因後移出 |

狀態更新規則：

- 暫緩項不能只寫「之後再看」。
- 暫緩項至少每完成一個里程碑檢查一次。
- 若一個暫緩項長期沒有回來條件，應重新判斷是否真的值得保留。
- 一旦恢復閱讀，就不要同時留在「暫緩中」和「問題索引待查」兩邊造成重複。

## 3. 暫緩優先級

暫緩也有優先級。它不是現在的閱讀順序，而是之後回來時的排序。

| 優先級 | 定義 | 例子 | 回來時機 |
|---|---|---|---|
| D1 | 高價值支線，完成主流程後應優先回來 | Table fixed / summary、Select remote、Modal confirm 細節 | 相關代表元件主流程完成後 |
| D2 | 中價值支線，進入專題章時回來 | popper 深入、async-validator 細節、完整 locale | 高階專題或支撐系統階段 |
| D3 | 低價值支線，只在遇到實際問題時回來 | assets 細節、logo、完整 examples wrapper | 有明確問題或落地需求時 |
| D4 | 暫不值得追 | minified 產物逐行閱讀、外部庫完整源碼 | 通常不回來，除非發生證據衝突 |

優先級判斷：

| 問題 | 若答案是「是」 | 建議 |
|---|---|---|
| 它會影響當前主流程嗎？ | 是 | 不暫緩，當場處理或放 `00-09` |
| 它會影響本階段 10 / 10 驗收嗎？ | 是 | D1 或直接處理 |
| 它只影響某個大型元件分支嗎？ | 是 | D1 / D2 |
| 它只是外部庫內部原理嗎？ | 是 | D2 / D4 |
| 它只是產物細節或展示資源嗎？ | 是 | D3 / D4 |

## 7. 第一輪預設暫緩區域

以下區域不是永遠不讀，而是第一輪不應該逐行深入。

| 暫緩區域 | 暫緩理由 | 第一輪可讀到哪裡 | 回來條件 |
|---|---|---|---|
| `node_modules/` | 外部依賴源碼量大，容易離開 View UI Plus 主線 | 只確認依賴名稱與用途 | 遇到外部庫行為阻塞時 |
| [`dist/`](../ViewUIPlus-SourceCode/dist/) minified 產物 | 產物可用來對照輸出，不適合第一輪逐行讀 | 只看入口、檔名、產物類型 | source 與 dist 出現證據衝突時 |
| 外部庫完整原理 | 例如 popper.js、async-validator、dayjs 內部實作不是本庫主線 | 只確認 View UI Plus 如何呼叫它 | 進入對應高階專題或 bug 追蹤時 |
| 所有語系檔逐一閱讀 | 語系檔數量多，初期只需理解載入與格式 | 讀 locale 入口與一兩個代表語系 | 進入國際化章節或遇到文案問題時 |
| 所有 examples 頁面逐一閱讀 | examples 是使用證據，不是第一閱讀主線 | 只讀當前元件對應 Demo | 分析某元件行為時 |
| 所有 tests 逐一閱讀 | tests 應用於代表元件對照，不必先全讀 | 先讀 Button、Select、DatePicker、Table、Message 等代表測試 | 進入測試反推規格章節時 |
| 所有 assets / iconfont 細節 | 多數是資源或樣式支撐，初期收益低 | 確認資產存在與被誰引用 | 樣式或打包資產出問題時 |
| 歷史設定細節 | Vue CLI 與 Vite 並存、舊設定的歷史原因不一定影響主流程 | 確認當前 dev / build 入口 | 工程章節需要釐清時 |
| 大型元件所有分支 | Select、DatePicker、Table 分支太多 | 第一輪只追主狀態、主事件、主渲染 | 主流程完成後逐支線恢復 |

## 8. 暫緩類型索引

用類型快速判斷暫緩策略。

| 類型 | 代表內容 | 暫緩理由 | 恢復方式 |
|---|---|---|---|
| 產物型 | `dist/`、minified JS、compiled CSS | 不適合第一輪理解原始設計 | 用 source 對照產物 |
| 外部庫型 | popper.js、async-validator、dayjs | 容易離開本庫主線 | 只追 View UI Plus 呼叫點 |
| 大型元件分支型 | Select remote、DatePicker time、Table fixed | 支線多，會打斷主流程 | 主流程完成後逐條恢復 |
| 歷史設定型 | Vue CLI / Vite 並存、舊 config | 需要時間背景，初期不阻塞 | 工程章節二輪再查 |
| 展示證據型 | examples wrapper、Demo 排版 | 不是規格本身 | 只查對應元件用法 |
| 覆蓋型 | 所有語系、所有 tests、所有 types | 全量讀成本高 | 先讀代表樣本 |
| 邊界案例型 | keyboard、allow-create、drag、resize | 初期不影響主閉環 | 進入該元件二輪精讀 |

## 9. 按學習階段整理暫緩項

### 9.1 第 0 階段：總覽與讀碼策略

| 暫緩項 | 暫緩理由 | 回來條件 |
|---|---|---|
| 補完整每個後續章節的詳細問題 | 00 章只建立方法，不提前塞入所有細節 | 讀到對應章節時 |
| 提前抽象所有高階設計模式 | 缺少代表元件證據，容易變成空泛總結 | 至少讀完 Button、Input、Form、Select、Message、Table |
| 為每個元件預先建立完整分析筆記 | 成本過高，且沒有源碼證據 | 讀到該元件時 |

### 9.2 第 1 階段：工程設定、依賴、建置與產物

| 暫緩項 | 暫緩理由 | 第一入口 | 回來條件 |
|---|---|---|---|
| `dist/viewuiplus.min.js` 逐行閱讀 | minified 產物不適合第一輪理解設計 | [`dist/viewuiplus.min.js`](../ViewUIPlus-SourceCode/dist/viewuiplus.min.js) | source 與 dist 出現不一致 |
| 完整 Rollup / Vite 每個選項深入 | 第一輪只需知道 library build 入口與輸出 | [`vite.config.js`](../ViewUIPlus-SourceCode/vite.config.js) | 工程章二輪或建置錯誤 |
| Gulp style build 每個 plugin 細節 | 第一輪只需理解 Less 到 CSS 的流程 | [`build/build-style.js`](../ViewUIPlus-SourceCode/build/build-style.js) | 樣式 build 出錯或進入工程專題 |
| Vue CLI 與 Vite 並存歷史原因 | 初期只需確認當前 dev / build 入口 | [`vue.config.js`](../ViewUIPlus-SourceCode/vue.config.js)、[`vite.config.dev.js`](../ViewUIPlus-SourceCode/vite.config.dev.js) | 工程章需要釐清歷史設定時 |

### 9.3 第 2 階段：入口、插件、註冊與全域配置

| 暫緩項 | 暫緩理由 | 第一入口 | 回來條件 |
|---|---|---|---|
| 每個全域命令式 API 的完整 instance 實作 | 第 2 階段先理解掛載，不追每個 service 內部 | [`src/index.js`](../ViewUIPlus-SourceCode/src/index.js) | 進入 Message / Notice / Modal 章節 |
| 每個 directive 的具體行為 | 入口章只需確認註冊方式 | `src/directives` | 進入 `19_src-directives_自訂指令` |
| 每個 component 的 install 後行為 | 入口章只追註冊鏈，不追元件內部 | [`src/components/index.js`](../ViewUIPlus-SourceCode/src/components/index.js) | 進入單一元件分析 |

### 9.4 第 3 階段：基礎元件與布局元件

| 暫緩項 | 暫緩理由 | 第一入口 | 回來條件 |
|---|---|---|---|
| 所有基礎元件逐一精讀 | 第一輪應精讀代表元件，略讀同類元件 | `src/components` | Button 模式建立後再按需求補 |
| Iconfont 完整字型與圖示清單 | 初期只需知道 Icon 如何使用 class / font | `src/styles/common/iconfont` | 樣式或 Icon 專題需要時 |
| Row / Col 所有響應式邊界 | 第一輪先理解 class 生成主線 | `src/components/row`、`src/components/col` | 進入布局專題二輪 |

### 9.5 第 4 階段：表單、輸入、資料展示與資料操作

| 暫緩項 | 暫緩理由 | 第一入口 | 回來條件 |
|---|---|---|---|
| Select remote / allow-create / keyboard / transfer 細節 | 先追 modelValue、option、visible、emit 主線 | [`src/components/select/index.js`](../ViewUIPlus-SourceCode/src/components/select/index.js) | Select 主流程筆記完成 |
| DatePicker 所有日期工具與 time spinner 細節 | 先追 picker、panel、parse / format、range 主線 | [`src/components/date-picker/index.js`](../ViewUIPlus-SourceCode/src/components/date-picker/index.js) | DatePicker 主流程筆記完成 |
| Table drag / resize / tree / expand / summary 完整分支 | 先追 columns、data、header、body、cell 主線 | [`src/components/table/index.js`](../ViewUIPlus-SourceCode/src/components/table/index.js) | Table 主流程圖完成 |
| async-validator 完整內部原理 | 先理解 View UI Plus 如何呼叫 validator | Form / FormItem | 表單架構專題或驗證 bug |

### 9.6 第 5 階段：導航、彈層、命令式 API、業務型元件

| 暫緩項 | 暫緩理由 | 第一入口 | 回來條件 |
|---|---|---|---|
| popper.js 完整定位演算法 | 第一輪只需知道 View UI Plus 如何傳 placement / visible | `base/popper.js`、Tooltip、Poptip | 進入彈層定位專題 |
| Modal scroll lock / mask / destroy 邊界 | 先分清普通 Modal 與 confirm service | [`src/components/modal/index.js`](../ViewUIPlus-SourceCode/src/components/modal/index.js) | Modal 主流程完成 |
| Upload / Transfer 所有邊界分支 | 屬於增強型元件，第一輪只追主流程 | `src/components/upload`、`src/components/transfer` | 基礎表單與資料元件完成後 |
| Login / Exception 視覺與業務細節 | 業務型元件應後期看封裝方式 | `src/components/login`、`src/components/exception` | 進入企業後台封裝章 |

### 9.7 第 6 階段：支撐系統

| 暫緩項 | 暫緩理由 | 第一入口 | 回來條件 |
|---|---|---|---|
| 所有 utils 逐函式精讀 | 先建立工具函式到元件的對照，不按檔案全讀 | [`src/utils`](../ViewUIPlus-SourceCode/src/utils) | 某元件主流程需要該工具 |
| 所有 mixins 一次性重構 | 先理解用途，不在閱讀階段立即重構 | [`src/mixins`](../ViewUIPlus-SourceCode/src/mixins) | 進入 Mixin 與 Composition API 重構章 |
| 所有 locale 語系檔逐一比對 | 初期只需理解 locale 入口與格式 | [`src/locale`](../ViewUIPlus-SourceCode/src/locale) | 國際化章二輪 |
| 所有 types 檔逐一核對 | 先對照代表元件 types | [`types`](../ViewUIPlus-SourceCode/types) | 型別宣告章或發現不一致 |
| 所有 tests 一次性全讀 | 先讀代表元件測試 | [`test/unit/specs`](../ViewUIPlus-SourceCode/test/unit/specs) | 測試反推規格章 |

### 9.8 第 7 與第 8 階段：高階設計與總結

| 暫緩項 | 暫緩理由 | 第一入口 | 回來條件 |
|---|---|---|---|
| 直接重構整個元件庫 | 沒有足夠代表元件證據前，重構方案容易失準 | `28` 到 `39` | 至少完成主要代表元件筆記 |
| 建完整迷你元件庫 | 需要前面階段累積 API、install、style、test 經驗 | `37_迷你元件庫實作` | 第 3 到第 6 階段完成 |
| 企業後台完整封裝方案 | 應先理解基礎元件、表單、彈層與命令式 API | `38_企業後台元件封裝實戰` | 高階設計章完成後 |

## 10. 大型元件暫緩分支

大型元件最容易讓第一輪失焦，應先建立分支索引。

### 10.1 Select

第一輪只追：

- `modelValue` 如何初始化與同步。
- option 如何註冊、顯示與選中。
- dropdown visible 如何控制。
- `update:modelValue` 與 `on-change` 何時觸發。

暫緩：

| 暫緩分支 | 暫緩理由 | 回來條件 |
|---|---|---|
| remote | 需要追非同步搜尋與 option 更新 | Select 主流程完成後 |

| keyboard navigation | 需要追焦點、hover、keydown 狀態 | visible 與 option 主流程完成後 |
| transfer / popper | 屬於彈層定位系統 | 進入彈層專題 |

### 10.2 DatePicker

第一輪只追：

- `picker.vue` 主入口。
- panel 如何切換。
- `modelValue` 如何 parse / format。
- 單值與 range 如何分流。
- `on-change`、`on-ok`、`update:modelValue` 如何觸發。

暫緩：

| 暫緩分支 | 暫緩理由 | 回來條件 |

## 11. 外部庫暫緩策略

外部庫要追到「View UI Plus 如何使用它」，不應一開始追完整庫原理。

| 外部庫 / 類型 | 第一輪要知道 | 暫緩內容 | 回來條件 |
|---|---|---|---|
| `async-validator` | Form / FormItem 何時呼叫驗證 | validator 內部規則解析完整流程 | 表單驗證專題二輪 |
| `popper.js` | Tooltip / Poptip / Select 如何傳 placement | 定位演算法與瀏覽器邊界完整處理 | 彈層定位專題 |
| `dayjs` | 日期格式化與 `$Date` 掛載關係 | dayjs 內部 plugin 與 parsing 細節 | DatePicker 日期處理出現問題 |
| Vue runtime | 元件如何使用 Vue API | Vue 內部實作 | 只有遇到框架層問題時 |
| Gulp / Vite / Rollup | 本專案如何配置它們 | 工具完整內部實作 | build 出錯或工程專題 |

判斷標準：

- 如果問題是「View UI Plus 何時呼叫它」，要追。
- 如果問題是「外部庫內部如何完整實作」，先暫緩。
- 如果外部庫行為影響本庫 bug 或測試，才恢復深入。

## 12. 產物與資源暫緩策略

產物、資源、字型、圖片常常有用，但第一輪不適合逐行讀。

| 區域 | 第一輪用途 | 暫緩內容 |
|---|---|---|
| [`dist/viewuiplus.min.js`](../ViewUIPlus-SourceCode/dist/viewuiplus.min.js) | 確認 JS 產物存在 | minified 逐行閱讀 |
| [`dist/viewuiplus.min.esm.js`](../ViewUIPlus-SourceCode/dist/viewuiplus.min.esm.js) | 對照 ES 產物 | 逐行反推 source |
| [`dist/styles/viewuiplus.css`](../ViewUIPlus-SourceCode/dist/styles/viewuiplus.css) | 對照 style build 輸出 | 逐條 selector 閱讀 |
| `dist/locale` | 確認語系輸出 | 所有語系逐一比對 |
| `assets` | 確認 logo / 圖片用途 | 圖片內容與展示細節 |
| iconfont | 確認字型被樣式引用 | 每個 icon glyph 細節 |

恢復條件：

- source 與 dist 出現不一致。
- build 產物缺失或樣式未生效。
- 發布、打包或資產路徑成為本輪問題。
- 進入 `05_dist_打包產物分析` 或 `27_assets_靜態資源` 的二輪精讀。

## 13. 從問題索引轉入暫緩

`00-09_問題索引.md` 中的問題，如果已判斷「現在不追，但日後有價值」，就應轉入本篇。

轉入前檢查：

| 檢查項 | 標準 |
|---|---|
| 問題是否已明確 | 能說出具體支線，不是模糊疑問 |
| 是否不阻塞當前主線 | 不處理也能完成目前章節 |
| 是否有日後價值 | 對二輪精讀、專題、仿寫或 bug 追蹤有幫助 |
| 是否有第一入口 | 之後回來可以直接打開 |
| 是否有回來條件 | 不會永遠停留在暫緩 |

轉入格式：

```md
| ID | 狀態 | 優先級 | 類型 | 暫緩項 | 所屬階段 | 暫緩理由 | 第一入口 | 回來條件 |
|---|---|---|---|---|---|---|---|---|
| D-YYYYMMDD-01 | 暫緩中 | D1 | 大型元件分支 | Table fixed left / right DOM 計算 | 第 4 階段 | 不影響 Table columns / data 主流程 | `src/components/table/table.vue` | Table 主流程圖完成後 |
```

## 14. 恢復閱讀流程

恢復暫緩項時，不要直接一頭鑽進去。先建立一張小任務卡。

```md
## 恢復閱讀任務卡

- 暫緩 ID：
- 本輪問題：
- 第一入口：
- 閱讀邊界：
- 已知證據：
- 這次仍不追的支線：
- 停止條件：
- 期望產出：
```

恢復閱讀流程：

```text
確認回來條件已達成
  -> 打開暫緩項第一入口
  -> 建立本輪問題
  -> 用 00-06 檢查表追蹤
  -> 若變成具體問題，轉入 00-09
  -> 若完成，回填對應章節
  -> 更新本篇狀態為已完成
```

恢復時的停止條件：

| 暫緩類型 | 可以停止的條件 |
|---|---|
| 大型元件分支 | 能畫出該分支入口、狀態、事件、輸出 |
| 外部庫 | 能說明 View UI Plus 如何呼叫外部庫，以及外部庫回傳結果如何被使用 |
| dist 產物 | 能說明 source 到產物的對照，不必逐行讀完 |
| examples | 能確認用法證據，不把 Demo 當完整規格 |
| tests | 能列出測試固定行為與缺口 |
| styles | 能把 class 對到 Less、變數、mixin 或 animation |

## 17. 常見不合格情況

| 不合格情況 | 問題 | 修正方式 |
|---|---|---|
| 只寫「之後再看」 | 沒有入口與回來條件 | 補暫緩理由、第一入口、回來條件 |
| 把不懂的問題都丟進暫緩 | 問題沒有被管理 | 先放 `00-09_問題索引.md` |
| 暫緩項太大 | 之後仍然無法開始 | 拆成小支線，例如 Table fixed、summary、tree 分開 |
| 暫緩理由不明 | 無法判斷是否該恢復 | 寫清楚它如何影響或不影響主線 |
| 沒有目前證據 | 下次會從零開始 | 至少記錄已確認入口與已讀到哪 |
| 永遠不檢查暫緩項 | 清單會失效 | 每完成一個里程碑檢查一次 |
| 恢復後不回填 | 知識無法沉澱 | 更新對應章節與問題索引 |
| 把 dist 當第一閱讀入口 | 容易從產物反推失焦 | 回到 source，再用 dist 對照 |
| 深入外部庫內部 | 離開 View UI Plus 主線 | 只追本庫如何呼叫外部庫 |

## 本節小結

暫緩閱讀清單的核心不是少讀，而是讓閱讀順序可控：

```text
主線先完成
  -> 支線先記錄
  -> 暫緩有理由
  -> 回來有入口
  -> 恢復有條件
  -> 完成有回填
```

只要每個暫緩項都能留下「為什麼現在不追」和「之後從哪裡回來」，第一輪讀碼就能保持清楚的主流程，第二輪也能精準補齊那些真正值得深入的細節。
