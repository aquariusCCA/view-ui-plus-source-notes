## 本節導讀

這篇是閱讀 View UI Plus 源碼時的「關鍵概念索引」。前面幾節已經建立地圖、路線、方法與元件分析模板；這一節負責把後續會反覆遇到的概念集中放在同一個入口，方便讀碼時快速定位。

本篇不追求把每個概念講到最深，而是回答三件事：

1. 這個概念在 View UI Plus 中大致負責什麼？
2. 第一次要從哪個源碼位置開始查？
3. 後續如果要深入，應該接到哪個章節或哪類元件？

本篇的定位是索引，不是替代後續章節。深入分析仍交給 `06` 之後的專題章與單一元件筆記。

## 建議回查情境

當你遇到以下情境，可以回到這篇：

- 看到一個名詞，例如 `prefixCls`、`globalProperties`、mixin、provide/inject，但不確定它在整個元件庫中的位置。
- 想快速知道某個概念應該先查哪個檔案。
- 想把單一元件分析連回整體架構。
- 想分辨一段源碼屬於註冊機制、全域配置、樣式系統、表單協作、命令式 API，還是型別 / 測試反推。
- 想整理後續章節時，確認每個概念應該放在哪個專題。

## 1. 概念總覽

先把 View UI Plus 的常見概念分成幾類：

| 類別 | 代表概念 | 第一入口 | 深入章節 |
|---|---|---|---|
| 工程入口 | package scripts、Vite library build、style build、locale build | `package.json` | `02`、`03`、`04`、`05` |
| 套件安裝 | `install`、`app.use`、`app.component`、`app.directive` | `src/index.js` | `06`、`07`、`08` |
| 元件暴露 | `components/index.js`、單元件 `index.js`、default export | `src/components/index.js` | `07`、`10` |
| 全域能力 | `$VIEWUI`、`globalProperties`、命令式 API、`$Date` | `src/index.js` | `08`、`31` |
| 元件內部 | props、emits、slots、data、computed、watch、methods、render | 單一 `.vue` | `11` 到 `18` |
| 樣式系統 | `prefixCls`、Less、variables、mixins、animation | `src/styles/index.less` | `23`、`33` |
| 共用協作 | mixin、provide/inject、utils、base service、directives | `src/mixins/`、`src/utils/`、`src/components/base/` | `19`、`20`、`21` |
| 對外契約 | types、examples、tests | `types/`、`examples/`、`test/` | `24`、`25`、`26`、`35` |
| 高階設計 | API 設計、受控狀態、彈層、命令式 API、表單架構 | 代表元件筆記 | `28` 到 `39` |

合理推論：讀單一元件時，通常會同時碰到多個概念。不要把它們拆散讀，應該用一條問題鏈把它們串起來。

## 2. 工程入口與建置概念

### 2.1 package scripts

已確認：[`package.json`](../ViewUIPlus-SourceCode/package.json) 中：

- `main` 指向 `dist/viewuiplus.min.js`。
- `typings` 指向 `types/index.d.ts`。
- `files` 包含 `dist`、`src`、`types`。
- `build` 依序執行 JS build、style build、locale build。

讀碼意義：

- `package.json` 是理解「發布套件長什麼樣」的入口。
- `scripts` 可以反推正式建置分成哪些線。
- `main` 和 `typings` 可以反推使用者安裝套件後拿到的 JS 與型別入口。

常見誤讀：

- 不要從 `dist/viewuiplus.min.js` 開始理解設計。`dist` 是產物觀察區，不是初期主讀入口。

### 2.2 Vite library build

已確認：[`vite.config.js`](../ViewUIPlus-SourceCode/vite.config.js) 的 library entry 是 `src/index.js`，輸出包含 UMD 與 ES 兩種格式。

讀碼意義：

```text
package.json build:prod
  -> vite build
  -> vite.config.js
  -> src/index.js
  -> dist/viewuiplus.min.js / viewuiplus.min.esm.js
```

合理推論：要理解 View UI Plus 的 JS 對外入口，第一個要讀的是 `src/index.js`。

### 2.3 style build

已確認：[`build/build-style.js`](../ViewUIPlus-SourceCode/build/build-style.js) 從 `src/styles/index.less` 編譯 CSS，輸出到 `dist/styles/viewuiplus.css`，並複製 iconfont 字型。

已確認：[`src/styles/index.less`](../ViewUIPlus-SourceCode/src/styles/index.less) 第一層 import 順序是：

```less
@import "./custom";
@import "./base";
@import "./mixins/index";
@import "./common/index";
@import "./animation/index";
@import "./components/index";
```

讀碼意義：

- 追樣式時，從元件 class 回到 component Less。
- 追全域樣式入口時，從 `src/styles/index.less` 回看變數、base、mixins、common、animation、components。

## 3. 套件安裝與入口概念

### 3.1 `install`

已確認：[`src/index.js`](../ViewUIPlus-SourceCode/src/index.js) 匯出 `install(app, opts = {})`。

`install` 目前包含幾類動作：

| 動作 | 已確認位置 | 讀碼意義 |
|---|---|---|
| locale 設定 | `opts.locale`、`opts.i18n` 進入 `localeFile.use` / `localeFile.i18n` | app 安裝時可設定語系 |
| 元件註冊 | 遍歷 `ViewUI` 並呼叫 `app.component` | 全域元件註冊入口 |
| 指令註冊 | 遍歷 `directives` 並呼叫 `app.directive` | 全域指令註冊入口 |
| `$VIEWUI` | 寫入 `app.config.globalProperties.$VIEWUI` | 保存全域尺寸、transfer、圖示與部分元件配置 |
| 命令式 API | 掛載 `$Message`、`$Notice`、`$Modal` 等 | 讓元件內可用 `this.$Message` 等呼叫 |
| `$Date` | 掛載 dayjs | 提供日期工具入口 |

待驗證：目前在 `src/index.js` 已看到 `if (install.installed) return;`，但本檔未讀到 `install.installed = true`。因此只能確認存在防重複安裝判斷，不能直接斷言它已完整防止重複安裝。

### 3.2 `app.component`

已確認：`install` 會遍歷 `ViewUI`，對每個 key 呼叫 `app.component(key, ViewUI[key])`。

讀碼路線：

```text
src/components/<component>/index.js
  -> src/components/index.js
  -> src/index.js import * as components
  -> ViewUI
  -> install
  -> app.component
```

讀碼意義：

- 想知道某個元件是否能被全域使用，不能只看 `.vue` 是否存在。
- 至少要確認它是否進入 `src/components/index.js`，以及是否被 `install` 的 `ViewUI` 遍歷到。

### 3.3 `app.directive`

已確認：[`src/index.js`](../ViewUIPlus-SourceCode/src/index.js) 只把 `directives` 物件中的項目全域註冊。

目前全域註冊入口包含：

- style 系列：`display`、`width`、`height`、`margin`、`padding`、`font`、`color`、`bg-color`
- `resize`
- `line-clamp`

已確認：[`src/directives/`](../ViewUIPlus-SourceCode/src/directives/) 中還存在 `clickoutside.js`、`transfer-dom.js`、`v-click-outside-x.js` 等檔案。

讀碼意義：

- 不能把 `src/directives/` 底下所有檔案都直接視為全域指令。
- 是否全域可用，要回到 `src/index.js` 的 `directives` 物件確認。
- 沒被 `install` 全域註冊的 directive，仍可能被某些元件局部 import 使用，需回到具體元件查證。

## 4. 元件匯出與註冊概念

### 4.1 `src/components/index.js`

已確認：[`src/components/index.js`](../ViewUIPlus-SourceCode/src/components/index.js) 是集中匯出清單，大量使用：

```js
export { default as Xxx } from './xxx';
```

讀碼意義：

- 它回答的是「哪些元件 / 服務被納入 View UI Plus 對外 API」。
- 讀單一元件時，這裡是確認對外暴露的必查位置。
- 對命令式服務，例如 `Message`、`Notice`、`Modal`，這裡也可能只是把 service object 對外暴露，不代表它是普通 `.vue` 元件。

### 4.2 單元件 `index.js`

單元件資料夾的 `index.js` 可能有不同角色：

| 類型 | 例子 | 讀法 |
|---|---|---|
| 簡單 re-export | `button/index.js` | 直接進入 `button.vue` |
| 子元件群組入口 | `tabs`、`breadcrumb`、`timeline` | 檢查父子元件如何一起匯出 |
| 命令式服務入口 | `message/index.js`、`modal/index.js` | 追 service method、instance、destroy |
| 底層支撐層 | `base/notification`、popper 相關檔 | 反查誰依賴它，不一定直接對外 |

合理推論：看到 `index.js` 時，第一步不是讀方法細節，而是判斷它是哪一種入口。

### 4.3 元件別名

已確認：[`src/index.js`](../ViewUIPlus-SourceCode/src/index.js) 的 `ViewUI` 物件除了展開 `components`，還補了 `iButton`、`iForm`、`iInput`、`iTable` 等別名。

讀碼意義：

- 某些元件可能同時能用 `Button` 與 `iButton` 形式註冊。
- 要確認別名範圍，應回到 `src/index.js` 的 `ViewUI` 物件，而不是只看 `src/components/index.js`。

### 5.1 `$VIEWUI`

已確認：`install` 會把 `$VIEWUI` 寫到 `app.config.globalProperties`，其中包含：

- `size`
- `capture`
- `transfer`
- `cell`
- `menu`
- `modal`
- `tabs`
- `select`
- `colorPicker`
- `cascader`
- `tree`
- `datePicker`
- `timePicker`

## 6. 命令式 API 概念

### 6.1 Message

已確認：[`src/components/message/index.js`](../ViewUIPlus-SourceCode/src/components/message/index.js) 不是普通 `.vue` 元件，而是 service object。

它包含：

- `info`
- `success`
- `warning`
- `error`
- `loading`
- `message`
- `config`
- `destroy`

已確認：`Message` 透過 `Notification.newInstance` 建立通知實例，並透過 `instance.notice` 新增消息。

讀碼流程：

```text
this.$Message.success(options)
  -> src/index.js globalProperties.$Message
  -> components.Message
  -> src/components/message/index.js
  -> getMessageInstance()
  -> Notification.newInstance()
  -> instance.notice()
```

讀碼重點：

- 追命令式 API 要看 instance 何時建立、是否重用、如何 remove / destroy。
- `config` 類方法通常會修改 defaults，而不是立即產生 UI。

### 6.2 Modal

已確認：[`src/components/modal/index.js`](../ViewUIPlus-SourceCode/src/components/modal/index.js) 會在 `Modal` 上補 `info`、`success`、`warning`、`error`、`confirm`、`remove` 等方法。

讀碼意義：

- `Modal` 同時可能是元件與命令式服務入口。
- 讀 `Modal.confirm()` 時，要追 `confirm(props)`、`getModalInstance()`、`instance.show(options)`、`remove()`。

合理推論：命令式 API 是 View UI Plus 讀碼中最容易誤把「元件」與「服務」混在一起的區域。分析時應優先使用 `00-04_元件分析模板.md` 中的命令式 API 模板。

## 7. 指令概念

### 7.1 style 指令

已確認：[`src/directives/style.js`](../ViewUIPlus-SourceCode/src/directives/style.js) 匯出多個 directive definition，例如 `display`、`width`、`height`、`margin`、`padding`、`font`、`color`、`bgColor`。

已確認：這些 directive 會在 `mounted` / `updated` 時改寫元素 style，並在 `unmounted` 時清理。

讀碼意義：

- style 指令屬於輕量 DOM 行為，不是元件 props。
- 它們是否全域可用，要回到 `src/index.js` 的 `directives` 物件確認。

### 7.2 resize 指令

已確認：[`src/directives/resize.js`](../ViewUIPlus-SourceCode/src/directives/resize.js) 使用 `element-resize-detector`，在 `mounted` 時 listen，在 `unmounted` 時 remove listener。

讀碼意義：

- 這類指令涉及 DOM listener 或第三方庫時，要特別看清理邏輯。
- 若後續遇到記憶體洩漏或 resize 觸發問題，可從 directive lifecycle 追起。

## 8. 樣式與 `prefixCls`

### 8.1 `prefixCls`

已確認：多個元件會在 `.vue` 中定義 `prefixCls`，例如 [`button.vue`](../ViewUIPlus-SourceCode/src/components/button/button.vue) 使用 `ivu-btn`。

Button 的 class 流程可以簡化成：

```text
const prefixCls = 'ivu-btn'
  -> computed classes
  -> render class
  -> src/styles/components/button.less
```

讀碼意義：

- `prefixCls` 是連接邏輯狀態與 Less selector 的橋。
- 看 class 時不能只記字串，要追它由哪些 props / computed 狀態控制。

### 8.2 Less 對照

已確認：樣式總入口是 [`src/styles/index.less`](../ViewUIPlus-SourceCode/src/styles/index.less)。

讀單一元件樣式時，建議順序：

```text
元件 prefixCls / computed class
  -> src/styles/components/<component>.less
  -> src/styles/mixins/
  -> src/styles/custom.less
  -> src/styles/animation/
```

待驗證：不是每個元件都一定有單一同名 Less 檔，也可能依賴 common、animation 或其他元件樣式。遇到找不到同名檔時，不要直接判定它沒有樣式。

## 9. 元件 API 與渲染概念

### 9.1 props / emits / slots

已確認：[`button.vue`](../ViewUIPlus-SourceCode/src/components/button/button.vue) 宣告 props、`emits: ['click']`，並在 render 中使用 default slot。

讀碼意義：

- props 是外部輸入，但不等於 UI 行為本身。
- emits 是對外通知，但不等於內部狀態已改變。
- slots 可能影響 class，例如 Button 的 `showSlot` 會影響 icon-only class。

### 9.2 computed / methods / render

以 Button 為例：

| 區塊 | 作用 |
|---|---|
| `classes` | 把 props / 狀態轉成 class |
| `tagName` | 判斷渲染成 `button` 或 `a` |
| `tagProps` | 補上 `href/target` 或原生 `type` |
| `handleClickLink` | emit click，並委託 link mixin 處理跳轉 |
| `render` | 把 tag、class、event、Icon、slot 組成實際輸出 |

讀碼意義：

```text
props
  -> computed
  -> methods / event
  -> render
  -> style
```

這條線是讀單一元件最常用的主流程。

## 10. mixin 概念

### 10.1 `mixins/form`

已確認：[`src/mixins/form.js`](../ViewUIPlus-SourceCode/src/mixins/form.js) inject `FormInstance` 與 `FormItemInstance`，提供：

- `itemDisabled`
- `handleFormItemChange(type, data)`

讀碼意義：

- 某些表單控件的 disabled 狀態不只來自自己的 `disabled` prop，也可能來自 Form。
- 某些 blur / change 驗證觸發點不在控件本身，而是透過 mixin 呼叫 FormItem。

### 10.2 `mixins/link`

已確認：[`src/mixins/link.js`](../ViewUIPlus-SourceCode/src/mixins/link.js) 提供：

- `to`
- `replace`
- `target`
- `append`
- `linkUrl`
- `handleOpenTo`
- `handleClick`
- `handleCheckClick`

已確認：Button mixin 了 `mixinsLink`。

讀碼意義：

- Button 的跳轉能力不是只在 `button.vue` 自己定義。
- 如果只讀 Button 的 props，會漏掉 `to`、`replace`、`target`、`append` 這些外部 API。

常見誤讀：

- 不要只看元件本身 props 就說 API 已完整。遇到 `mixins` 必須回頭查 mixin 補了什麼。

## 11. provide / inject 概念

### 11.1 Form / FormItem

已確認：[`form.vue`](../ViewUIPlus-SourceCode/src/components/form/form.vue) provide `FormInstance`。

已確認：[`form-item.vue`](../ViewUIPlus-SourceCode/src/components/form/form-item.vue) inject `FormInstance`，並 provide `FormItemInstance`。

表單協作可以簡化成：

```text
Form provide FormInstance
  -> FormItem inject FormInstance
  -> FormItem provide FormItemInstance
  -> Input / Select 等控件透過 mixins/form inject FormItemInstance
  -> 控件 blur / change
  -> FormItem.validate()
```

讀碼意義：

- provide / inject 是父子協作與跨層溝通的重要入口。
- 表單驗證不是單一元件內部行為，而是 Form、FormItem、控件、mixin、AsyncValidator 協作。

### 11.2 大型元件

合理推論：除了 Form，Select、DatePicker、Menu、Tabs 等大型互動或父子協作元件也可能使用 provide / inject、子元件註冊或上下文共享。讀這類元件時，應先建立子模組地圖，再追主流程。

## 12. 表單驗證概念

已確認：[`form.vue`](../ViewUIPlus-SourceCode/src/components/form/form.vue) 維護 `fields`，並提供：

- `resetFields`
- `validate`
- `validateField`
- `addField`
- `removeField`

已確認：[`form-item.vue`](../ViewUIPlus-SourceCode/src/components/form/form-item.vue) 使用 `async-validator`，並在 `mounted` 時把自己加入 Form 的 fields，在 `beforeUnmount` 時移除。

表單驗證主流程：

```text
FormItem mounted
  -> FormInstance.addField(this)
  -> 使用者操作控件
  -> mixins/form.handleFormItemChange('blur' 或 'change')
  -> FormItem.formBlur / formChange
  -> FormItem.validate(trigger)
  -> AsyncValidator
  -> validateState / validateMessage
  -> Form emit on-validate
```

讀碼重點：

- `rules` 可能來自 Form，也可能來自 FormItem。
- `trigger` 會過濾要執行的 rule。
- `validateState` 會影響 class 與錯誤訊息渲染。
- `resetField` 會回寫 model，需要小心看 `initialValue` 的來源。

讀碼意義：

## 14. 型別概念

### 14.1 `types/index.d.ts`

已確認：[`types/index.d.ts`](../ViewUIPlus-SourceCode/types/index.d.ts) 匯出 `viewuiplus.components`，並宣告：

- `ViewUIPlusGlobalOptions`

讀碼意義：

- 它是對外元件型別索引。
- 若實作有某個元件，但 types 沒有，或 types 多出實作不明的項目，應標記為「待驗證」或「證據衝突」。

## 15. examples 與 tests 概念

### 15.1 examples

已確認：[`examples/main.js`](../ViewUIPlus-SourceCode/examples/main.js) 從 `../src/index` 載入 View UI Plus，並呼叫 `app.use(ViewUIPlus)`。

讀碼意義：

### 15.2 tests

已確認：[`test/unit/specs/`](../ViewUIPlus-SourceCode/test/unit/specs/) 中存在部分代表性測試，例如 Button、Select、Table、Message、DatePicker 等。

- 若測試和實作、types、examples 不一致，應標記為「證據衝突」。

## 16. 常見概念混淆

| 容易混淆的概念 | 正確讀法 |
|---|---|
| 有 `.vue` 檔就等於對外可用 | 還要確認 `src/components/index.js` 與 `src/index.js install` |
| `src/components/<name>/index.js` 一定是元件本體 | 先判斷它是 re-export、群組入口、service，還是底層支撐層 |
| `$emit` 等於狀態改變 | `$emit` 只是通知外部，狀態是否改變要看外部 props / model 回流 |
| props 就是完整 API | mixin、slots、events、public methods、types 也可能補 API |
| class 字串等於樣式已確認 | 要追到 Less selector 與樣式入口 |
| directives 資料夾裡的指令都全域可用 | 只有 `src/index.js` 的 `directives` 物件會被全域註冊 |
| types 是執行時真相 | types 是對外契約，需要和實作、examples、tests 對照 |
| examples 是完整規格 | examples 是使用方式證據，不是完整行為規格 |
| 命令式 API 和普通元件一樣讀 | 要追全域掛載、service 方法、instance、remove / destroy |
| provide/inject 只是一種傳值方式 | 在表單和大型協作元件中，它通常是架構主線 |

## 17. 概念串聯路線

### 17.1 從 `app.use` 到全域元件

```text
examples/main.js
  -> app.use(ViewUIPlus)
  -> src/index.js install
  -> import * as components
  -> ViewUI
  -> app.component
  -> 使用者可在 template 使用元件
```

### 17.2 從 Button prop 到樣式

```text
Button props
  -> computed classes
  -> render class
  -> prefixCls = 'ivu-btn'
  -> src/styles/components/button.less
  -> src/styles/index.less
```

### 17.3 從表單控件事件到驗證

```text
Input / Select blur 或 change
  -> mixins/form.handleFormItemChange
  -> FormItem.formBlur / formChange
  -> FormItem.validate(trigger)
  -> AsyncValidator
  -> validateState / validateMessage
  -> Form on-validate
```

### 17.4 從 `$Message.success` 到 DOM 通知

```text
this.$Message.success(options)
  -> globalProperties.$Message
  -> components.Message
  -> message/index.js
  -> getMessageInstance
  -> Notification.newInstance
  -> instance.notice
```

### 17.5 從型別到實作驗證

```text
types/<component>.d.ts
  -> src/components/<component> 實作
  -> examples/routers/<component>.vue
  -> test/unit/specs/<component>.spec.js
  -> 記錄一致 / 不一致 / 待驗證
```

## 本節小結

View UI Plus 的關鍵概念可以濃縮成一條主線：

> `package.json` 與 build 設定決定套件如何輸出，`src/index.js` 決定安裝時如何註冊元件、指令與全域 API，`src/components/index.js` 決定哪些元件進入對外 API，單一元件用 props、狀態、事件、渲染與樣式完成行為，mixins、provide/inject、utils、directives、locale、types、examples、tests 則分別提供共用能力、上下文協作、工具支撐、DOM 行為、國際化、對外契約、使用證據與行為證據。

後續讀碼時，如果不確定某個概念屬於哪條線，先回到本篇第 0 節快速查找表，再依照對應入口展開。這樣可以避免在元件細節、樣式、型別與測試之間來回跳轉卻沒有留下可驗證的閱讀鏈。
