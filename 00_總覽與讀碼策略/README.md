# 00_總覽與讀碼策略

本章是閱讀 View UI Plus 源碼前的操作手冊。目標不是立刻深入每個元件，而是先建立穩定的源碼地圖、閱讀路線、分析模板與回查方式，避免後續讀碼時只是在檔案之間跳轉，卻沒有形成可累積的理解。

建議先讀 `00-01` 到 `00-03`，建立整體方向；接著使用 `00-04` 到 `00-07` 作為每次分析元件或模組時的固定工具；最後用 `00-08` 到 `00-10` 管理學習進度、問題與暫緩範圍。

|---|---|

| 00-01_源碼總地圖.md | 建立 View UI Plus 原始碼的第一層地圖，快速辨識 `src`、`build`、`dist`、`examples`、`types`、`test`、`assets` 等目錄的責任邊界。 |
| 00-02_讀碼路線.md | 規劃整體閱讀路線，包含工程建置線、入口安裝線、元件註冊線、元件分析線、樣式系統線、測試反推線與高階設計線。 |
| 00-03_讀碼方法.md | 固定每次讀源碼的操作順序：先找入口，再追依賴，接著分析狀態、事件、渲染、樣式、測試與對外 API。 |

| 00-05_關鍵概念索引.md | 整理閱讀 View UI Plus 時會反覆遇到的核心概念，例如插件安裝、元件匯出、指令註冊、`globalProperties`、`prefixCls`、mixin、provide/inject、命令式 API、表單驗證與國際化。這份筆記只做索引與定位，深入分析交給後續專題章。 |

## 使用原則

- 先讀主線，再補支線：優先理解安裝入口、元件註冊、全域配置、樣式入口與代表性元件。
- 每次只追一條問題鏈：例如「Button 如何被註冊」或「Message 如何命令式呼叫」，避免同時追太多分支。
- 每篇筆記都要留下源碼位置、角色定位、關鍵流程、可仿寫點與待查問題。
- 00 章只負責建立方法與索引，不把所有細節提前塞進總覽章；深入分析放到後續對應章節。
