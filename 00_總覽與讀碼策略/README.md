# 00_總覽與讀碼策略

[返回根目錄 README](../README.md)

## 本章定位

本章是 View UI Plus 源碼閱讀的入口章。它先不急著深入單一元件，而是建立後續讀碼會反覆使用的三件事：

1. 專案地圖：知道 `package.json`、`build/`、`src/`、`types/`、`examples/`、`test/` 各自負責什麼。
2. 讀碼方法：知道如何從入口、註冊、定義、生效位置一路追到樣式、型別、範例與測試。
3. 維護機制：知道如何記錄問題、暫緩支線、檢查追蹤完整度，避免讀碼過程越追越散。

後續每一章都會回到本章的規則：先確認主線，再追證據，最後把結論整理成可回查的筆記。

## 建議閱讀順序

第一輪建議順讀 `00-01` 到 `00-10`。如果時間有限，可以先讀到 `00-06`，確保已有基本讀碼地圖、路線、方法、模板與檢查表，再回來補 `00-07` 到 `00-10` 的選讀與維護機制。

1. 先讀 [00-01_源碼總地圖](./00-01_源碼總地圖.md)，建立整個專案的位置感。
2. 再讀 [00-02_讀碼路線](./00-02_讀碼路線.md)，選擇工程、入口、元件、樣式、測試或高階設計主線。
3. 接著讀 [00-03_讀碼方法](./00-03_讀碼方法.md)，統一每次追源碼的操作順序。
4. 用 [00-04_元件分析模板](./00-04_元件分析模板.md)、[00-05_關鍵概念索引](./00-05_關鍵概念索引.md)、[00-06_追蹤檢查表](./00-06_追蹤檢查表.md) 建立後續章節的筆記格式與驗收標準。
5. 最後用 [00-07_元件選讀策略](./00-07_元件選讀策略.md)、[00-08_學習里程碑](./00-08_學習里程碑.md)、[00-09_問題索引](./00-09_問題索引.md)、[00-10_暫緩閱讀清單](./00-10_暫緩閱讀清單.md) 管理進度、問題與暫緩支線。

## 本章讀完後應能做到

- 能說明 View UI Plus 從 `package.json`、`vite.config.js`、`src/index.js` 到 `src/components/index.js` 的主入口關係。
- 能把一個元件拆成入口、註冊、對外 API、內部狀態、事件流、渲染、樣式、型別、範例與測試幾條證據線。
- 能分辨「已確認」、「合理推論」、「待驗證」、「證據衝突」這幾種筆記狀態。
- 能使用元件分析模板產出後續章節的單一元件分析筆記。
- 能判斷哪些源碼分支應該第一輪精讀，哪些應該暫緩到第二輪。
