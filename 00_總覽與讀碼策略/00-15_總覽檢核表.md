## 1. 這份檢核表的用途

這份筆記不是用來「看完打勾」而已，而是用來判斷你是否真的完成了 View UI Plus 的系統性讀碼。

你讀 View UI Plus 的最終目標不是：

- 知道每個元件怎麼用
- 把官方文件看過一次
- 把源碼從頭翻到尾
- 把 AI 的解釋複製成筆記

而是要做到：

1. 能說明一套 Vue 3 元件庫如何被安裝、註冊、配置與使用。
2. 能讀懂單一元件的 API、狀態、事件、樣式與型別設計。
3. 能讀懂跨元件機制，例如 Form/FormItem、Message/Modal、Locale、PrefixCls、Directive。
4. 能將 View UI Plus 的設計轉化成自己的企業後台封裝能力。
5. 能仿寫一套迷你元件庫，並說明每個設計決策。
6. 能把讀碼筆記變成日後查詢、複習、面試、重構與專案實作的知識庫。

---

## 2. 使用方式

建議你在四個時間點使用這份檢核表。

### 2.1 開始讀碼前

用來確認你是否已經理解整體目標、閱讀順序與輸出標準。

重點不是全部打勾，而是知道自己接下來要往哪裡走。

### 2.2 每讀完一個章節後

用來檢查該章節是否只是「看過」，還是已經真的形成了可回查的筆記與可仿寫的理解。

### 2.3 每週複盤時

用來確認本週是否有產出：

- 一份完整筆記
- 一張流程圖
- 一個仿寫元件
- 一個企業封裝案例
- 一組可回答的問題

### 2.4 完成整套 View UI Plus 筆記後

用來檢查你是否已經從「讀源碼」進階到「能設計自己的元件庫」。

## 3. 檢核狀態標記規則

建議使用以下狀態標記，不要只用單純的完成或未完成。

| 狀態 | 意義 |
|---|---|
| `[ ]` | 尚未開始 |
| `[~]` | 已閱讀，但理解不完整 |
| `[x]` | 已完成，且有筆記或產出 |
| `[R]` | 需要回查或重讀 |
| `[A]` | 已完成，且能用 AI 追問後通過驗證 |
| `[P]` | 已轉化成仿寫或企業封裝產出 |

範例：

```md
- [x] 我知道 `app.use(ViewUIPlus)` 會觸發 `install`
- [R] 我還不確定 Message 的實例建立流程
- [P] 我已經仿寫 MiniMessage
```
