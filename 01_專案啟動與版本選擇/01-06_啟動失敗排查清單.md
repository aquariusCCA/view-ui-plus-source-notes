# 01-06_啟動失敗排查清單

> 所屬章節：`01_專案啟動與版本選擇`  
> 筆記定位：View UI Plus 本體源碼啟動失敗時的分層排查手冊  
> 適用目標：遇到 `npm install`、`npm run dev2`、`npm run dev`、`npm run build` 失敗時，可以依序定位問題  
> 建立日期：2026-05-04  
> 建議前置筆記：  
> - `01-04_本地開發環境準備_Node_pnpm_Vue版本.md`  
> - `01-05_從零Clone與啟動ViewUIPlus源碼.md`

---

## 1. 本篇要解決的問題

讀 View UI Plus 源碼時，最常見的阻礙不是一開始看不懂元件，而是：

```text
專案跑不起來
依賴裝不起來
dev server 起不來
build 失敗
瀏覽器畫面空白
CSS 沒有載入
改了源碼但畫面沒變
```

這些問題如果沒有分層排查，很容易變成：

```text
看到錯誤就 Google
看到建議就亂升級
npm 改 pnpm
Node 16 改 Node 20
Vue 改最新版
Vite 改最新版
最後專案更亂
```

所以本篇的核心目標是：

> **建立一套 View UI Plus 啟動失敗時的標準排查流程，讓你能先定位錯誤發生在哪一層，再決定怎麼處理。**

---

## 2. 本篇結論先講

View UI Plus 本體目前同時包含：

```text
Vue CLI 啟動路線：npm run dev
Vite 啟動路線：npm run dev2
完整打包路線：npm run build
樣式打包路線：npm run build:style
語言包打包路線：npm run build:lang
```

而且本體依賴中存在較舊的工程工具：

```text
@vue/cli-service ~4.5.0
vite ^2.6.4
gulp ^4.0.2
less ^2.7.3
less-loader ^4.1.0
typescript ^3.3.4000
vue ^3.2.47
vue-template-compiler ^2.6.14
```

所以第一輪排查建議原則是：

```text
先不要升級依賴
先不要改 package manager
先不要把 Vue / Vite / TypeScript 升到最新版
先不要直接 npm audit fix --force
先固定 Node 16.20.x + npm
先把 install / dev2 / dev / build 分開測
```

一句話：

> **先還原專案當前工具鏈，再做問題定位；不要一遇到錯誤就升級整個工程。**

---

## 3. 先確認：你現在卡在哪一階段？

排查前先不要急著修。  
請先判斷錯誤發生在哪一階段。

| 階段 | 指令 | 常見問題 |
|---|---|---|
| Clone 階段 | `git clone` | 網路、權限、路徑過長、Git 設定 |
| 版本確認階段 | `git status` / `git rev-parse HEAD` | 分支不對、tag 不對、commit 沒記錄 |
| Node 切換階段 | `nvm use` / `node -v` | Node 版本不對、nvm 沒生效 |
| 依賴安裝階段 | `npm install` | peer dependency、lockfile、registry、node-gyp |
| Vite 啟動階段 | `npm run dev2` | Vite config、ESM、port、依賴解析 |
| Vue CLI 啟動階段 | `npm run dev` | Webpack、OpenSSL、loader、Vue CLI |
| Build 階段 | `npm run build` | Vite build、Gulp style、language build |
| 瀏覽器階段 | 開 local url | 白畫面、console error、CSS missing |
| Hot reload 階段 | 修改源碼 | 改錯檔案、cache、不是跑本地源碼 |
| 樣式階段 | CSS 顯示 | less、dist styles、class prefix、import 路徑 |

排查時先問：

```text
錯誤是發生在 install 前、install 中、dev server、build，還是瀏覽器？
```

這個問題比錯誤訊息本身更重要。

---

## 4. 標準排查總流程

遇到任何啟動問題，先照這個流程走：

```text
Step 1：保存完整錯誤訊息
Step 2：確認目前路徑是否在 ViewUIPlus 根目錄
Step 3：確認 git branch / commit
Step 4：確認 Node / npm 版本
Step 5：確認是否混用 lockfile
Step 6：確認使用的是 npm，不是 pnpm / yarn 混用
Step 7：刪除 node_modules 後重裝
Step 8：分別測試 npm run dev2 和 npm run dev
Step 9：分段測試 npm run build:prod / build:style / build:lang
Step 10：把結果寫入 error-log.md
```

---

## 5. 第一層：確認基本環境

### 5.1 確認目前所在目錄

很多錯誤其實只是因為你不在專案根目錄。

執行：

```bash
pwd
ls
```

Windows PowerShell：

```powershell
Get-Location
Get-ChildItem
```

你應該看到：

```text
package.json
src
build
types
dist
vite.config.js
vite.config.dev.js
```

如果沒有看到 `package.json`，代表你可能不在 ViewUIPlus 根目錄。

---

### 5.2 確認 Git 狀態

```bash
git status
git branch --show-current
git rev-parse HEAD
git remote -v
```

目的：

```text
確認你讀的是哪個分支
確認你是否有本地修改
確認你是否能回溯 commit
確認 remote 是否是官方 ViewUIPlus
```

---

### 5.3 確認 package.json 版本

```bash
node -p "require('./package.json').name"
node -p "require('./package.json').version"
node -p "require('./package.json').scripts"
```

目前 View UI Plus 本體應該是：

```text
name：view-ui-plus
version：1.3.24
```

如果不是，先確認你是否 clone 到錯的專案，或是否切到不同 tag / branch。

---

## 6. 第二層：確認 Node / npm

### 6.1 查版本

```bash
node -v
npm -v
```

建議本體第一輪使用：

```text
Node 16.20.x
npm 隨 Node 16 附帶版本即可
```

---

### 6.2 常見 Node 版本錯誤

如果你用 Node 20 / 22 / 24 跑舊工具鏈，可能出現：

```text
ERR_OSSL_EVP_UNSUPPORTED
error:0308010C:digital envelope routines::unsupported
webpack crypto error
loader error
dependency build failed
```

處理方式：

```bash
nvm install 16.20.2
nvm use 16.20.2
node -v
npm -v
```

然後重新安裝依賴。

---

### 6.3 nvm use 沒生效怎麼辦？

症狀：

```bash
nvm use 16.20.2
node -v
```

結果還是其他版本。

可能原因：

```text
終端機沒有重開
PATH 順序錯誤
Windows 同時安裝了官方 Node installer 和 nvm-windows
VS Code terminal 沒重新載入環境變數
```

處理：

```text
1. 關掉所有 terminal
2. 重新開 PowerShell / Git Bash
3. 再執行 nvm use
4. 再確認 node -v
5. 必要時移除全域 Node installer
```

---

## 7. 第三層：確認 package manager

### 7.1 第一輪建議只用 npm

View UI Plus 本體第一輪建議：

```bash
npm install
npm run dev2
npm run dev
npm run build
```

不要一開始用：

```bash
pnpm install
yarn install
```

原因：

```text
本體 package.json scripts 使用 npm 串接
舊工具鏈可能假設 npm 的 node_modules 結構
pnpm 的嚴格依賴結構可能暴露隱式依賴問題
```

---

### 7.2 確認是否混用 lockfile

檢查：

```bash
ls *lock*
```

Windows PowerShell：

```powershell
Get-ChildItem *lock*
```

可能看到：

```text
package-lock.json
pnpm-lock.yaml
yarn.lock
```

原則：

```text
同一個專案不要混用多種 lockfile
```

如果你曾經用 pnpm 裝過，後來要回 npm，建議清理：

```bash
rm -rf node_modules
rm -f pnpm-lock.yaml
npm install
```

Windows PowerShell：

```powershell
Remove-Item -Recurse -Force node_modules
Remove-Item -Force pnpm-lock.yaml
npm install
```

---

## 8. 第四層：依賴安裝失敗

### 8.1 `npm install` 失敗先看什麼？

請先記錄：

```bash
node -v
npm -v
npm config get registry
```

然後看錯誤類型：

```text
ERESOLVE
ETIMEDOUT
ECONNRESET
EACCES
node-gyp
Cannot find module
postinstall failed
peer dependency conflict
```

---

### 8.2 `ERESOLVE unable to resolve dependency tree`

常見原因：

```text
npm 7+ 對 peerDependencies 檢查較嚴
舊專案依賴宣告與新版 npm 解析規則衝突
```

第一步不要升級依賴。

可嘗試：

```bash
npm install --legacy-peer-deps
```

但要記錄到 `install-log.md`：

```md
| 是否使用 legacy-peer-deps | 是 |
| 原因 | ERESOLVE peer dependency conflict |
| Node | 16.20.2 |
| npm | 待填 |
```

---

### 8.3 `ETIMEDOUT` / `ECONNRESET`

常見原因：

```text
網路不穩
公司代理
registry 連線問題
VPN 問題
npm registry 暫時異常
```

檢查：

```bash
npm config get registry
npm ping
```

可設定官方 registry：

```bash
npm config set registry https://registry.npmjs.org/
```

如果在公司網路：

```bash
npm config get proxy
npm config get https-proxy
```

---

### 8.4 `EACCES` 權限錯誤

常見原因：

```text
全域 npm 權限問題
node_modules 權限異常
用管理員裝過依賴
資料夾權限被鎖
```

處理：

```text
1. 不要把專案放在 Program Files
2. 不要放在需要管理員權限的目錄
3. 刪除 node_modules
4. 重新用一般使用者權限安裝
```

Windows 建議放：

```text
D:\code\view-ui-plus-study
```

---

### 8.5 `node-gyp` 錯誤

可能原因：

```text
某些舊依賴需要原生編譯
Python / C++ build tools 缺失
Node 版本不相容
```

第一優先處理：

```text
先切 Node 16.20.x
先確認是不是某個舊套件被迫編譯
不要先升級整個依賴樹
```

如果仍需處理 Windows build tools，再另行處理。

---

## 9. 第五層：`npm run dev2` 失敗

`dev2` 對應：

```bash
vite --config vite.config.dev.js
```

代表這條路線是 Vite 開發啟動。

---

### 9.1 `vite: command not found`

可能原因：

```text
依賴沒安裝成功
node_modules/.bin 不存在
vite 沒在 devDependencies 中安裝
使用了錯誤 package manager
```

檢查：

```bash
npm list vite
ls node_modules/.bin
```

Windows：

```powershell
npm list vite
Get-ChildItem node_modules/.bin
```

處理：

```bash
rm -rf node_modules
npm install
npm run dev2
```

---

### 9.2 `Cannot find module xxx`

可能原因：

```text
依賴安裝不完整
pnpm 嚴格依賴導致找不到隱式依賴
package-lock / node_modules 狀態不一致
```

處理順序：

```bash
rm -rf node_modules
npm install
npm run dev2
```

如果你之前用 pnpm：

```bash
rm -rf node_modules
rm -f pnpm-lock.yaml
npm install
```

---

### 9.3 Vite port 被占用

錯誤可能類似：

```text
Port 5173 is already in use
Port xxxx is already in use
```

處理：

```text
1. 關閉其他 dev server
2. 找出占用 port 的 process
3. 或臨時指定其他 port
```

可用：

```bash
npx vite --config vite.config.dev.js --port 5174
```

但如果要長期改，應該在 config 中處理，並記錄修改。

---

### 9.4 ESM / CommonJS 錯誤

可能訊息：

```text
Cannot use import statement outside a module
require() of ES Module not supported
```

排查方向：

```text
Node 版本是否過新或過舊
Vite 版本是否被升級
某個依賴是否被 npm update 改掉
是否誤用了最新 Vite 文件套到舊 Vite 2 專案
```

處理：

```text
先回到 Node 16 + 原 package.json 依賴
不要升級 Vite
```

---

## 10. 第六層：`npm run dev` 失敗

`dev` 對應：

```bash
vue-cli-service serve
```

代表這條路線是 Vue CLI / Webpack 開發啟動。

---

### 10.1 `vue-cli-service: command not found`

可能原因：

```text
依賴沒有安裝
@vue/cli-service 安裝失敗
node_modules/.bin 不存在
```

檢查：

```bash
npm list @vue/cli-service
```

處理：

```bash
rm -rf node_modules
npm install
npm run dev
```

---

### 10.2 OpenSSL / Webpack crypto 錯誤

常見錯誤：

```text
error:0308010C:digital envelope routines::unsupported
ERR_OSSL_EVP_UNSUPPORTED
```

常見於：

```text
新 Node + 舊 Webpack / Vue CLI 工具鏈
```

建議處理：

```bash
nvm use 16.20.2
rm -rf node_modules
npm install
npm run dev
```

不建議第一步就用：

```bash
set NODE_OPTIONS=--openssl-legacy-provider
```

因為那只是繞過，不是還原合適工具鏈。

---

### 10.3 loader 相關錯誤

可能訊息：

```text
this.getOptions is not a function
Module build failed
less-loader error
css-loader error
vue-loader error
```

排查方向：

```text
loader 版本是否被升級
webpack 版本是否與 loader 不相容
Node 版本是否太新
是否執行過 npm update
```

處理：

```text
1. 回到 package.json 原本版本
2. 刪除 node_modules
3. 使用 Node 16
4. npm install
```

---

### 10.4 `vue-template-compiler` 相關錯誤

你可能會疑惑：

```text
View UI Plus 是 Vue 3 專案，為什麼 package.json 裡有 vue-template-compiler？
```

這通常和舊 Vue CLI / loader 工具鏈歷史包袱有關。

處理原則：

```text
不要一看到 vue-template-compiler 就刪掉
不要直接改成 @vue/compiler-sfc 最新版
先照 package.json 安裝並啟動
後續在 package 分析章再研究原因
```

---

## 11. 第七層：`npm run build` 失敗

完整 build 對應：

```bash
npm run build
```

它會依序執行：

```bash
npm run build:prod
npm run build:style
npm run build:lang
```

所以 build 失敗時，不要只看總錯誤，要分段跑。

---

### 11.1 分段定位

```bash
npm run build:prod
npm run build:style
npm run build:lang
```

然後建立表格：

| 指令 | 是否成功 | 失敗類型 |
|---|---|---|
| `npm run build:prod` | 待填 | Vite build / JS |
| `npm run build:style` | 待填 | Gulp / Less / CSS |
| `npm run build:lang` | 待填 | 語言包 / Vite config |

---

### 11.2 `build:prod` 失敗

`build:prod` 對應：

```bash
vite build
```

排查方向：

```text
Vite config
Rollup build
ES module import
外部依賴 external
輸出格式
Node 版本
依賴版本是否被改動
```

先看：

```bash
npm list vite
node -v
```

---

### 11.3 `build:style` 失敗

`build:style` 對應：

```bash
gulp --gulpfile build/build-style.js
```

這條路線常見問題：

```text
gulp 找不到
less 編譯錯誤
gulp-less 錯誤
gulp-clean-css 錯誤
路徑問題
Windows 路徑分隔符問題
CSS 壓縮錯誤
```

檢查：

```bash
npm list gulp
npm list gulp-less
npm list less
npm list gulp-clean-css
```

處理：

```text
1. 確認 Node 版本
2. 確認 gulp 依賴存在
3. 確認 build/build-style.js 存在
4. 不要先升 gulp 或 less
5. 看具體是哪個 less 檔案報錯
```

---

### 11.4 `build:lang` 失敗

`build:lang` 對應：

```bash
vite build --config build/vite.lang.config.js
```

排查方向：

```text
build/vite.lang.config.js 是否存在
locale 目錄是否存在
語言包 export 是否正常
Vite 版本是否正確
```

檢查：

```bash
ls build
ls src/locale
npm run build:lang
```

---

### 11.5 build 成功但 dist 不完整

如果 build 沒報錯，但你找不到預期檔案，要檢查：

```text
dist 是否被清掉
build scripts 是否只產生部分檔案
package.json files 是否包含 dist / src / types
styles 是否放在 dist/styles
types 是否本來就不是 build 產生
```

檢查：

```bash
ls dist
ls dist/styles
ls types
```

---

## 12. 第八層：瀏覽器白畫面

### 12.1 先看 DevTools Console

白畫面不要先回 terminal，先開瀏覽器 DevTools：

```text
Console
Network
Elements
Sources
```

Console 常見錯誤：

```text
Cannot read properties of undefined
Failed to resolve component
Failed to fetch dynamically imported module
Uncaught SyntaxError
CSS MIME type error
404 resource not found
```

---

### 12.2 Network 看資源是否 404

檢查：

```text
JS 是否載入
CSS 是否載入
圖片 / icon 是否載入
字體是否載入
source map 是否報錯
```

如果 CSS 404，後續要追：

```text
樣式 import 路徑
dist/styles 是否存在
Vite dev config 是否處理 CSS
publicPath / base 是否設定錯誤
```

---

### 12.3 Elements 看 DOM 是否有渲染

如果 DOM 有內容，但畫面沒樣式：

```text
大概率是 CSS 沒載入
```

如果 DOM 幾乎沒有內容：

```text
大概率是 JS runtime error 或 mount 失敗
```

---

## 13. 第九層：CSS 沒有載入

### 13.1 症狀

```text
Button 看起來像原生 button
Input 沒有 View UI Plus 樣式
Layout 沒有排版
Icon 不顯示
Modal 沒有遮罩或位置錯誤
```

---

### 13.2 排查方向

檢查：

```text
是否有 import CSS
dist/styles/viewuiplus.css 是否存在
樣式是否由 Less build 產生
class 是否有 prefix
瀏覽器 Network 是否載入 CSS
CSS 是否被覆蓋
```

---

### 13.3 對使用者測試專案

如果是你自己的測試專案，請確認 `main.ts`：

```ts
import ViewUIPlus from 'view-ui-plus'
import 'view-ui-plus/dist/styles/viewuiplus.css'
```

如果只 `app.use(ViewUIPlus)`，但沒引入 CSS，元件功能可能正常，但樣式會不完整。

---

### 13.4 對本體源碼

如果是本體 source 啟動，則要看：

```text
vite.config.dev.js
examples 入口
style import
src/styles
build style script
```

不要直接把使用者專案的 CSS 引入方式套到本體 source。

---

## 14. 第十層：修改源碼但畫面沒有變

### 14.1 可能原因

```text
你改的不是 dev server 正在使用的檔案
瀏覽器 cache
Vite / Vue CLI HMR 沒觸發
source 和 dist 混淆
改到 npm package，不是本地 source
dev2 和 dev 使用不同入口
```

---

### 14.2 確認方式

在你認為會被執行的檔案中加：

```js
console.log('[study] file loaded')
```

如果瀏覽器 console 沒看到，表示：

```text
這個檔案不是目前路線的入口
```

---

### 14.3 dev 和 dev2 要分開看

你必須分清楚：

```text
npm run dev  使用 Vue CLI 路線
npm run dev2 使用 Vite 路線
```

不同路線可能使用不同 config、入口、alias 或 examples。

所以修改無效時，要先確認：

```text
我現在跑的是 dev 還是 dev2？
我改的檔案是否被這條路線使用？
```

---

## 15. Windows 專屬排查

### 15.1 路徑太深

症狀：

```text
安裝依賴奇怪失敗
找不到檔案
node_modules 路徑問題
```

建議：

```text
D:\code\view-ui-plus-study
```

不要：

```text
C:\Users\xxx\Desktop\很多層\很多層\View UI Plus 學習資料
```

---

## 18. 不建議的錯誤處理方式

遇到錯誤時，以下操作不要一開始就做。

### 18.1 不要直接 `npm audit fix --force`

這可能會大量升級依賴，導致：

```text
Vue CLI / Webpack / loader 版本不相容
Vite 版本跳太多
TypeScript 規則變嚴
Less 編譯壞掉
```

---

### 18.2 不要直接升級 Vue

不要一開始就：

```bash
npm install vue@latest
npm install @vue/compiler-sfc@latest
```

因為你的目標是讀 View UI Plus 當前版本，不是重構它到最新 Vue。

---

### 18.3 不要直接升級 Vite

本體目前是舊版 Vite 工具鏈。  
你自己的新測試專案可以用 Vite latest，但不要把最新 Vite 文件直接套回 View UI Plus 本體。

---

### 18.4 不要直接刪掉 vue-template-compiler

即使這看起來像 Vue 2 工具，仍可能是舊 Vue CLI / loader 工具鏈中的一部分。

第一輪不要先刪。

---

### 18.5 不要同時改很多東西

錯誤排查時一次只改一件事：

```text
只切 Node
只重裝依賴
只改 registry
只跑 dev2
只跑 dev
只跑 build:style
```

如果一次改十個地方，你就不知道是哪個改動解決了問題。

## 20. 快速定位表

| 錯誤訊息關鍵字 | 優先懷疑 | 優先處理 |
|---|---|---|
| `ERESOLVE` | peer dependency | `npm install --legacy-peer-deps` |
| `ERR_OSSL_EVP_UNSUPPORTED` | 新 Node + 舊 Webpack | 切 Node 16 |
| `vue-cli-service not found` | 依賴未裝好 | 重裝 npm 依賴 |
| `vite: command not found` | Vite 未安裝 / node_modules 壞 | 重裝依賴 |
| `Cannot find module` | 依賴缺失 / pnpm 嚴格依賴 | 回 npm 重裝 |
| `less-loader` | loader / webpack 相容 | 不升級，切 Node 16 |
| `gulp` | style build | 檢查 gulp / less / build-style |
| `Port is already in use` | port 被占用 | 關閉 process / 換 port |
| `Failed to resolve component` | 元件註冊 / alias / import | 檢查入口與註冊 |
| `CSS 404` | 樣式路徑 | 檢查 CSS import / dist |
| 白畫面 | runtime error | 看 browser console |
| 改源碼沒反應 | 改錯入口 / HMR | 加 console 確認路線 |
