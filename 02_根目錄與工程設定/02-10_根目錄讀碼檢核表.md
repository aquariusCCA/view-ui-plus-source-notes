## 0. 本篇先給結論

這篇不是新增一堆新的工程知識，而是幫你把 `02_根目錄與工程設定` 這一整章轉成一份可以反覆使用的檢核表。

你讀完 `02-01` 到 `02-09` 之後，應該不只是「知道 View UI Plus 根目錄有哪些檔案」，而是要具備以下能力：

1. 看得出哪些資料夾是**元件庫源碼**；
2. 看得出哪些資料夾是**開發展示與 Demo**；
3. 看得出哪些資料夾是**打包產物**；
4. 看得出哪些檔案控制**本地開發模式**；
5. 看得出哪些檔案控制**正式打包流程**；
6. 看得出哪些設定是**型別宣告與編輯器支援**；
7. 看得出哪些設定是**程式碼規範**；
8. 看得出哪些設定控制**npm 發布邊界**；
9. 看得出哪些設定和 **Git / CI / 線上預覽**有關；
10. 能判斷某些設定是「目前主線仍在用」還是「歷史遺留」。

這份檢核表的核心價值是：

> **把根目錄從「一堆檔案」變成「工程系統地圖」。**

### 4.3 設定檔讀碼三問

每看到一個根目錄設定檔，請問自己三個問題：

```text
1. 誰會讀這個檔案？
2. 什麼時機會讀這個檔案？
3. package.json scripts 或平台設定中，還有沒有入口會觸發它？
```

範例：

| 檔案 | 誰會讀？ | 什麼時機？ | 是否主線？ |
|---|---|---|---|
| `vite.config.js` | Vite | `npm run build:prod` / `vite build` | 是 |
| `vite.config.dev.js` | Vite | `npm run dev2` | 是，但屬於第二開發入口 |
| `vue.config.js` | Vue CLI | `npm run dev` | 是，且線上預覽也可能使用 |
| `.travis.yml` | Travis CI | Travis 觸發 CI 時 | 可疑，需檢查是否仍有效 |
| `tslint.json` | TSLint | 有 TSLint 入口時 | 可疑，需檢查 scripts |

### 5.2 Vue CLI 開發模式檢核

## 12. 判斷「主線設定」與「歷史遺留」的檢核表

```text

```

## 20. 本篇總結
